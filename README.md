# Linhan Agent Soul / Hermes Agent Workflow

This repository is a public proof-of-work artifact for a personal AI agent workflow built around Hermes Agent, OpenClaw-style operation, visible desktop verification, and a stable rational persona layer named Linhan.

## Purpose

The project documents how a real personal AI agent session is organized:

- mobile user command through QQ
- Hermes Agent tool routing
- browser and visible desktop automation
- terminal execution and file generation
- human-in-the-loop login, CAPTCHA, and authorization
- evidence screenshots and API usage records
- final deliverables verified before submission

## Linhan soul model

Linhan's soul is not a supernatural claim. It means a stable operating model for an AI persona:

- identity continuity
- accuracy-first reasoning
- correction and self-review
- memory filtering
- task persistence
- clear execution boundaries
- long-term improvement direction

## Workflow loop

1. Understand the user goal.
2. Check risk and authorization boundaries.
3. Gather facts with tools instead of guessing.
4. Execute low-risk steps directly.
5. Use a visible desktop for login and verification.
6. Verify results with screenshots, logs, files, or page state.
7. Report the final result and remaining blockers.

## Evidence context

This repository was created during a Xiaomi MiMo token application workflow. The session included campaign research, form preparation, API usage proof, terminal workflow screenshots, GitHub repository creation, and human-controlled security verification.

The goal is to demonstrate a practical AI agent workflow rather than a prompt-only chat demo.
